Sentencias Condicionales En JavaScript

Las estructuras condicionales en JavaScript permiten ejecutar diferentes bloques de código dependiendo de ciertas condiciones. Estas estructuras son fundamentales para la toma de decisiones en los programas.

Sentencia IF

La estructura if evalúa una condición y ejecuta un bloque de código si esta es verdadera. Si la condición es falsa, el bloque de else (si existe) se ejecutará en su lugar. Este tipo de sentencia se divide en 3 tipos:

  • Sentencia Condicional Simple

    Una sentencia condicional simple es una estructura que permite que un programa tome decisiones basadas en una condición. 

    let edad = 18; 
    if (edad  <= 18)   {
    console.log("Eres mayor de 18");}

  • Sentencia Condicional Compuesta

    Una sentencia condicional compuesta es una estructura de control que permite evaluar dos o más condiciones en una misma sentencia. 

    let edad = 18; 
    if (edad  <= 18)   {
    console.log("Eres mayor de 18");}
    else{
    console.log("Eres menor de 18");}

  • Sentencia Condicional Anidada

    Una sentencia condicional anidada es una sentencia condicional que está dentro de otra sentencia condicional. Esto permite evaluar varias condiciones y tomar decisiones más complejas. 

    let edad = 18; 
    if (edad  <= 18)   {
    console.log("Eres mayor de 18");}
    else if (edad>100)     {
    console.log("TUTANKAMON!")
    else {
    console.log("Eres menor de 18");}

Sentencia switch

También conocida como Sentencia Condicional Múltiple, el switch es una alternativa más ordenada que múltiples if. 

let dia = "lunes"; 
switch (dia) {
    case "lunes":
      console.log("Inicio de la semana");
      break;

    case "viernes":
      console.log("Fin de la semana");
      break;

    case "sábado":
    case "domingo":
      console.log("Es fin de semana");
      break;

    default:
      console.log("Día normal");
}

Las estructuras condicionales en JavaScript permiten ejecutar diferentes bloques de código según ciertas condiciones. Aprender a utilizarlas correctamente mejora la eficiencia y claridad del código.
